Convio URL parameters

URL Parameter What's it do?
AddInterest Addsthe Interest (specified by ID) to the logged in constituent's record.
autologin A value of "true" indicates that when a user clicks on the link from an Email message, they should be automatically be logged in.
df_preview Sets a donation form to preview mode, allowing the test credit card (4111 1111 1111 1111) to be used. Useful when you need to get to a form's thank you page and don't want to use your own credit card.
mfc_popup A value of "true" indicates the page should be displayed in a "pop-up" wrapper.
NEXTURL The URL to redirect the browser to after the user takes some action. Most common usecase is UserLogin. In Survey, it overrides the Survey Submitted function
pgwrap A value of "n" indicates the page should not be rendered inside a page wrapper.
pw_id Overrides the within a specific pagewrapper Id and version. If there are multiple security categories or multi-center, then you'll also need to use s_AffiliateSetCatId=1
RemoveInterest Removes the Interest (specified by ID) to the logged in constituent's record.
s_Affiliate Sets the Affiliate session variable, which is used by multi-affiliate sites.
s_AffiliateSecCatId Sets the security category (specified by ID) in the user's session. Often used in conjunction with pw_id to push a particular page wrapper into the user's session.
s_debug - Enables debug mode - good for troubleshooting anything. Setting this to true will also make it so that HTML comments are included above and below any reusables on a page indicating the pagename of the reusable.
s_debug_pc Set to pc2 or classic to force either type of participant center in your session
s_jsfirebuglite Loads Firebug Lite on the page, and every subsequent page in the session. Extremely useful for browsers that don't natively have Firebug plugins, like IE or mobile browsing.
s_promoCode Sets the specified promo code in the user's session. Promo codes can be used to display special ticket levels for Calendar/Ticketed Events or participation types for TeamRaiser events.
s_src Actively sets the source code in the user's session. If the user is new or a new record is created, this will double as their Origin Source Code.
s_subsrc Sets the sub-source code in the user's session. If the user is new or a new record is created, this will double as their Origin SubSource Code.
scid Sets the Center (specified by the Center's security category ID) in the user's session for Multi-Center clients.
set.foo See https://www.blackbaud.com/files/support/helpfiles/luminate-online/help/luminateonline.html#../Subsystems/donation_management/Content/Concepts/Admin_Online_Giving_Management_DonationForms_HiddenInputs.html for more info
